How an AI agent builds a real website

1. Research & structure

The agent browses competitor sites, docs, and inspiration — similar to how founders research manually. This aligns with guidance from Google to build for AI agents as first‑class users [searchenginejournal.com].

2. Generate code & assets

Super writes HTML, CSS, and copy, editing files directly. Unlike chat‑only tools such as ChatGPT, Gemini, or Grok, this happens inside a real filesystem.

3. Push to GitHub

The agent logs into GitHub, commits changes, and pushes updates — mirroring workflows used in modern hosting tutorials [hostinger.com].

4. Deploy & publish

Super navigates hosting dashboards (Vercel, Netlify, WordPress.com) and deploys. WordPress now explicitly supports agent‑driven publishing [techcrunch.com].

5. Iterate with computer-use cache

Every login flow, repo structure, and deployment path is remembered in a computer-use cache, so repeated updates cost less time and effort than one‑off runs.
